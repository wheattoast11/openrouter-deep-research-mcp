# v2.0 User Journeys: From Simple to "Magic"

This document illustrates the progression from basic usage to advanced agentic workflows, demonstrating the power of v2.0's new capabilities.

## Journey 1: Developer Monitoring Dependencies

### Level 1: Basic (v1.x style)
```javascript
// Manual research
await client.callTool({
  name: 'agent',
  arguments: {
    query: 'Are there security vulnerabilities in @electric-sql/pglite 0.2.17?',
    async: false
  }
});
```

### Level 2: Proactive (v2.0)
```javascript
// Set up continuous monitoring
await client.getPrompt({
  name: 'continuous_query',
  arguments: {
    query: '@electric-sql/pglite security vulnerabilities and updates',
    cronExpression: '0 */6 * * *' // Every 6 hours
  }
});

// Connect WebSocket for proactive alerts
const ws = new WebSocket('ws://localhost:3008/mcp/ws?token=TOKEN');
ws.on('message', (data) => {
  const msg = JSON.parse(data);
  if (msg.type === 'temporal.monitor_update' && msg.payload.newResults > 0) {
    alert('⚠️ New PGlite security information discovered!');
  }
});
```

### Level 3: "Magic" (v2.0 with knowledge graph)
```javascript
// One-time setup: Learn about your dependencies
await client.getPrompt({
  name: 'summarize_and_learn',
  arguments: {
    url: 'https://github.com/electric-sql/pglite/blob/main/package.json'
  }
});

// Query the knowledge graph any time
const deps = await client.callTool({
  name: 'query_graph',
  arguments: { entity: 'PGlite', maxHops: 2 }
});

// Knowledge graph automatically tracks:
// - PGlite version history
// - Related packages
// - Known vulnerabilities
// - Update patterns
```

**Result**: From manual checks → Continuous monitoring → Intelligent, context-aware dependency management

## Journey 2: Researcher Following a Field

### Level 1: Basic
```javascript
// Weekly manual research
await client.callTool({
  name: 'agent',
  arguments: {
    query: 'Latest developments in quantum computing'
  }
});
```

### Level 2: Scheduled Briefings
```javascript
// Schedule weekly briefings
await client.callTool({
  name: 'schedule_action',
  arguments: {
    cronExpression: '0 9 * * 1', // Monday 9 AM
    action: {
      type: 'research',
      query: 'Latest quantum computing breakthroughs (past 7 days)',
      costPreference: 'high' // Important topic, use best models
    }
  }
});

// Daily summary via magic prompt
await client.getPrompt({
  name: 'daily_briefing',
  arguments: { limit: 20 }
});
```

### Level 3: Evolving Knowledge Base
```javascript
// Knowledge graph accumulates:
// - Key researchers and institutions
// - Important papers and citations
// - Relationships between concepts
// - Temporal evolution of the field

// Query accumulated knowledge
const quantumGraph = await client.callTool({
  name: 'query_graph',
  arguments: { entity: 'Quantum Error Correction', maxHops: 3 }
});

// Unified retrieval combines:
// - Graph: Direct entity/relationship matches
// - Vector: Semantically similar past research
// - BM25: Keyword matches in reports

const insights = await client.callTool({
  name: 'retrieve',
  arguments: {
    query: 'quantum supremacy claims and verification methods'
  }
});

// Returns synthesized view across all knowledge sources
```

**Result**: From periodic queries → Automated tracking → Intelligent knowledge accumulation and synthesis

## Journey 3: Business Intelligence Analyst

### Level 1: Ad-Hoc Queries
```javascript
await client.callTool({
  name: 'agent',
  arguments: {
    query: 'Compare market share of cloud providers Q3 2025'
  }
});
```

### Level 2: Continuous Monitoring
```javascript
// Monitor competitors
await client.getPrompt({
  name: 'continuous_query',
  arguments: {
    query: 'AWS, Azure, and GCP market share and pricing changes',
    cronExpression: '0 0 * * *' // Daily
  }
});

// Morning briefing for executive team
await client.callTool({
  name: 'schedule_action',
  arguments: {
    cronExpression: '0 8 * * 1-5', // Weekdays 8 AM
    action: {
      type: 'briefing'
    }
  }
});
```

### Level 3: Strategic Intelligence Platform
```javascript
// Knowledge graph maps competitive landscape
const landscape = await client.callTool({
  name: 'query_graph',
  arguments: { entity: 'Amazon Web Services', maxHops: 3 }
});

// Graph automatically links:
// - Companies → Products → Pricing → Market Share
// - Temporal trends captured in metadata
// - Competitive relationships mapped

// Steer agent focus in real-time during research
ws.send(JSON.stringify({
  type: 'agent.steer',
  payload: { new_goal: 'Focus on serverless offerings comparison' }
}));

// Agent adapts mid-task and provides targeted analysis
```

**Result**: From reactive analysis → Continuous intelligence → Strategic, real-time competitive insights

// - Best practices

## Key Insights

### Progression Pattern

1. **Simple**: Direct tool calls (like v1.x)
2. **Automated**: Scheduled actions and continuous monitoring
3. **Intelligent**: Knowledge graph + bidirectional steering
4. **"Magic"**: System anticipates needs and proactively assists

### The "Magic" is in the Integration

No single feature is magical. The magic emerges from:
- **Temporal** + **WebSocket** = Proactive notifications
- **Knowledge Graph** + **Retrieval** = Context-aware answers
- **Magic Prompts** + **Scheduling** = Zero-friction workflows
- **Bidirectional** + **Streaming** = Real-time collaboration

### Design Philosophy

> "The best interface is invisible. Complexity should be hidden, intelligence should be obvious."

v2.0 hides:
- Multi-step orchestration
- Entity extraction pipelines
- Cron management
- WebSocket lifecycle

v2.0 reveals:
- Simple commands with powerful results
- Proactive insights when relevant
- Growing knowledge over time
- Real-time responsiveness
